# User Story

\> As a Halo acount owner I want to be able to resolve autogenerated fence overlaping with either edit of new fence or leaving of creation flow so that I can delete and replace old fences with new one.

# Acceptance criteria

| AC | Description | iOs design | Android design | Fence preview screen |
|---|---|---|---|---|
| ME04-US80-AC01 | When user Confirms the selected position of the fence creation before autogeneration the app should:Get the updated list of the closest 20 existing fences in users account.NoteThis is needed so that the app checks the overlapping with the 20 closest fences to the fence position chosen by the user. |  |  |
| ME04-US80-AC02 | If during autogeneration the app generates a fence that overlaps at least one of the existing fences in the user's account, then the app should:Display 'Fence preview with the Edit New Fence main action' screen with displayed autogenerated fence on it. |
| ME04-US80-AC03 | 'Fence preview with the replacing main action' screen should consist of the following elements:'Back' button;'Confirm Fence' screen header;Map + 'Compass' button;'Question mark' button;'Map Settings' button;Existing fence(s) that overlap autogenerated one;Autogenerated fence with warning area;Explanation text about fences overlapping;'Cancel' button'Edit New Fence' button; |
| ME04-US80-AC04 | The following elements should behave the same as on [Outdated] ME04-US61. Fence preview with warning area step:'Back' button;'Confirm Fence' screen header;Map + 'Compass' button;'Question mark' button;'Map Settings' button.See ME04-US61-AC03 – ME04-US61-AC08. |
| ME04-US80-AC05 | The app should display the Existing fence(s) that overlap the autogenerated one with:dark red border;transparent red filling. |
| ME04-US80-AC06 | The app should display an autogenerated fence: in the location chosen by the user on the ME04-US58. Fence location positioning step.with the warning area inside that calculated according to the already implemented algorithm;with the map centered on the fence so it is fully visible;hovering the existing overlapping fences. |
| ME04-US80-AC07 | The 'Explanation text about fences overlapping' should be the following:'You have overlapping fences. To continue please edit the new fence.' |
| ME04-US80-AC08 | If I click the 'Edit New Fence' button, then the app should:Display the 'Edit Mode' screen (requirements to the screen are described in a separate User Story: ME04-US63. Fence edit mode step) |
| ME04-US80-AC09 | If I click the 'Cancel, then the app should:Reset new fence creation.Open the My Map screen with the collapsed Find Card.NoteIt is expected the user will click on fences on the Map that they want to delete. |## Mobile analytics
