The API to save the fence is POST /geo-fence/addThe fields ActivityType = Active, and PublicVisibilityType = Private

Points are the edited points on the edit screenIf we moved here from the Confirm Fence screen, this is the warning zone (points generated by the Auto-generate fences)

If the backend returns ErrorCode = 3001 and InvalidOperationType = 103 - it means that the user has no option to add a fence because of the subscriptionDetails on what to display are described here - (NA) ME19-FE00-US03. Subscription сheck for potentially restricted features.

If the backend returns ErrorCode = 3001 and InvalidOperationType = 104 - This means we can no longer add the fence for technical reasons.In this case, M126 Communication error is displayed.

If the backend returns ErrorCode = 3001 and InvalidOperationType = 19 - It means that such a name already existsThere is also a separate endpoint to check name uniqueness - PUT /geo-fence/check-name-uniqueness - it accepts the name and id of the fence (will be null). Can be useful if validators are used

The Analytics field in the request will be filled in a separate store (NA) ME04-F14. Fence Analytics in Operational DBs.
