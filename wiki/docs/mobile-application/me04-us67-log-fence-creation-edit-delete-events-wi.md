User Story Acceptance criteria

# User Story

\> As a Halo Product Owner I want to be able to have in Azure AppInsights the data related the way user created the fence and if they edit, edit name or delete it late so that I could understand how often users edit autogenerated fence and how many fences at all bazed on cadastral data.

# Acceptance criteria

| AC | Description |
|---|---|
| ME04-US67-AC01 | Log to the Azure AppInsights the fence creation events containing the following parameters:Date+timeUser IDFence IDFence auto-generation approach: Simple square/AutoOverlapping after autogeneration: None/Water/Road/Other fencesWere there any edits during creation after auto-generation? (yes/no)NotesThe 'Overlapping after autogeneration' parameter can be implemented in any way that is considered easier by the developer (can be even 4 separate parameters).The 'Were there any edits during creation after auto-generation?' parameter is needed for corner cases when the algorithm autogenerates fence overlapping road and/or water or other fences. |
| ME04-US67-AC02 | Log to the Azure AppInsights the fence borders edit events containing the following parameters:Date+timeUser IDFence IDList of viewed warnings/errors names with qty of seen times |
| ME04-US67-AC03 | Log to the Azure AppInsights the fence name edit events containing the following parameters:Date+timeUser IDFence IDFence Name BeforeFence Name After |
| ME04-US67-AC04 | Log to the Azure AppInsights the fence delete events containing the following parameters:Date+timeUser IDFence ID |
