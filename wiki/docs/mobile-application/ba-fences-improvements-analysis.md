# Zendesk insights

1. Questions about using the house as a wall.
2. Questions about creating fences on hills.
3. Not everybody understands what 20 fences means (at the same time), and probably that fences may be deleted.
4. Not everybody understands the definition of fence, there was a user who supposed that 1 fence=1 part between fence posts.
5. Questions regarding size of fence.
6. Seems that for some users it is confusing that they should have a virtual fence where they already have a physical fence.
7. We don't explain users with only 15 ft around that they need to draw fences on neighbors' territory. We should help VS limit them.
8. Too big required distance between fences (20-30 ft).
9. Fence within a fence:
  1. Pool.
  2. Some part of the territory when a user is at home and not at home.
  3. Some farm dogs have access everywhere, some have to limited territory.

10. Different fences for different dogs (seems relevant to fix issues that was considered as fence within a fence),
11. Partially overlapping fences (for example for different dogs can be the same warning zone).
12. Possibility to turn on off/on fences separately.

## Improvements

- Until we have a fence within a fence - suggest users use a beacon if they try to overlap fences instead of telling them that it is forbidden.
- Revision of errors order displaying (when I tried to put the 2nd post too near to 1st one - the app displayed to me the "minimal 3 posts" error VS "at least 6 ft error" )
- Shouldn't we reconsider default emergency feedback from static and notify the user that it might make sense to have it?
- Display how many unused fence posts are left instead of pop-up after adding the 3rd pin.
- Return blinking of the first post - so user understand that they need to close it. Add the finish button - so it closes automatically. -*The most popular fence error pop-up*
- If users mostly use drawing VS collar creation - gather feedback on why they choose drawing, so we can understand how we can help them and improve creation with collar.
- In case we will implement the possibility of turning on/off collars individually - track usage and suggest removing rarely used ones.
- If reached 20 - suggest removing some of them.Suggest the list of fences -*Seem not really a required feature, because only 4 people have 20 fences.*

## New features

- Different fences for different dogs (notify if dog within a fence but for some reason deactivated)
- Possibility to turn off/on fences separately.
- Autogenerated fence
- Shared fences (reuse friend's fence)
- Unlimited number of fences (store all the fences on the account). We can sync/download only closest to the user. Or we can have 20 enabled and unlimited created fences.
- Add some collars grouping tool(travel, home with me, home without me, walking OR temporary VS usual - add expiration date when adding temporary). Especially if we allow to add more than 20 fences.
- Fence within a fence.
- Partially overlapping fences.
