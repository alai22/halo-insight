# User story

\> As Halo app account owner, I want to choose location of my fence creation, so the app autogenerate it where I need it.

# Acceptance criteria

| AC | Description | Design |
|---|---|---|
| ME04-US06-AC01 | If at least one of the following conditions is met:NO property lines data was found,Autogenerated fence has NO building(s) inside,Autogenerated fence is too big,Fence geodata validation error between the app and the backend during autogeneration,then the app should:Open the 'Edit Fence' screen.Display autogenerated simple square fence (see details below)Display the 'Not supported location' toast. | Figma |
| ME04-US06-AC02 | The 'Edit Fence' screen for simple square after autogeneration should work the same to 'Edit Screen' after 'Confirm Fence Step' within the 'Add Fence' flow (see (NA) ME04-US08. Add Fence: Step 4. Edit Fence (optional)) |
| ME04-US06-AC03 | The app should display the simple square fence following the requirements:Fence shape: Square;Area: 1500 m2;Center of the fence: In the place where user placed the 'Fence Location Positioning' pin on the 'Set Fence Location' step;Default 18th zoom level. |
| ME04-US06-AC04 | Simple square area 1500 m2 should be manageable via config. |
| ME04-US06-AC05 | Autogenerated fence is too big = a fence based on cadastral data where the registered size is bigger than 50000 m2 (~12.355269 acres). |
| ME04-US06-AC06 | The property registered size threshold of 50000 m2 is manageable via config. |Tech details
