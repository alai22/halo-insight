| HALO-6157 - MOB [RI]: ME04-US51. Open closed fences Closed |# User story

\> As a user I want to be able to open a closed fence so that I could add new fence posts to it.

# Acceptance criteria

| AC | Description | iOS UI design / Notes | Android UI design / Notes |
|---|---|---|---|
| IOS/Android implementation statuses | IOS TO DO | ANDROID TO DO |
| ME04-US51-AC01 | Users should be able to open a closed fence when they:create a new fence with fingercreate a new fence with GPSedit an existing fence with fingeredit an existing fence with GPS | - | - |
| ME04-US51-AC02 | When the user deletes a fence post of a closed fence, the adjacent to the deleted post borders should also be removed and the fence should open. | - | - |
| ME04-US51-AC03 | When the user opens a fence when creating or editing the fence with GPS, the "Move" button on the toolbar should be replaced with the "Add (GPS) button. | - | - |
| ME04-US51-AC04 | When the user opens a fence, the 'Next' button should disappear from the navigation bar. | - | - |
| ME04-US51-AC05 | After the user opens a fence, one of the outer fence posts should be selected and the other should become the first post of the fence and begin to pulse. | - | - |
| ME04-US51-AC06 | If a fence is opened and the user deletes a fence post, the selected fence post should be removed and the fence should auto-fill between the two adjacent to the deleted posts. | - | - |
| ME04-US51-AC07 | If a user opens a fence and then taps Undo, the app shall re-close the fence as it was. If the user then taps 'Redo', the app shall re-open the fence again. | - | - |
